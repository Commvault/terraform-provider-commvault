# commvault_cloudconnection (Resource)

 Use the commvault_cloudconnection resource type to create or delete a cloud connection in the CommCell environment. 

## Example Usage

**Create AWS Cloud Connection with all accounts and specific regions with identity center owner detection**

```hcl
resource "commvault_cloudconnection" "org_aws_1" {
  name               =  “AWSOrgConnection”
  startdiscoveryjob  = "true"
  connectiontype     = "OrganizationLevel"
  cloudtype          = "aws"
  enablediscovery    = "true"   # optional: whether discovery should run; set as needed

  # Cloud-specific configuration block (AWS)
  cloudspecificconfiguration {
    aws {
      regions         = “us-east-1, us-east-2”
      iamroleaccountid = “xxxxxxxxxxxxx”

      organizationconfiguration {
        enableownerdetection = “true”

        # Optional owner detection configuration (only needed when enableownerdetection = "true")
        # Uncomment and adapt if you use identity center owner detection
        ownerdetectionconfiguration {
           identitycenterregion = "us-east-2"
           ownerpermissionsets = ["Admin", "PowerUser"]
        }

        content {
          discoverallaccounts = “true”
          
        } # end content
      } # end organizationconfiguration
    } # end aws
  } # end cloudspecificconfiguration
 
}
```

**Create AWS Cloud Connection with specifc accounts and all regions with no identity center owner detection**

```hcl
resource "commvault_cloudconnection" "org_aws_1" {
  name               =  “AWSOrgConnection”
  startdiscoveryjob  = "true"
  connectiontype     = "OrganizationLevel"
  cloudtype          = "aws"
  enablediscovery    = "true"   # optional: whether discovery should run; set as needed

  # Cloud-specific configuration block (AWS)
  cloudspecificconfiguration {
    aws {
      regions         = “default”
      iamroleaccountid = “xxxxxxxxxxxxx”

      organizationconfiguration {
        enableownerdetection = “false”

        content {
          discoverallaccounts = “false”
          # Explicit accounts mapping (if discoverallaccounts = "false")
          # This provider schema expects 'accounts' as a set of objects.
          # Add one accounts { ... } block per account in your org you want to enumerate.
         accounts {
            name = "account-1"
            id    = "xxxxxxxxxxxx"
            email = ops@company.com
          }	
          
        } # end content
      } # end organizationconfiguration
    } # end aws
  } # end cloudspecificconfiguration
 
}
```
